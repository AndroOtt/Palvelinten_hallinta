##    Chacon and Straub 2014: Pro Git, 2ed: 1.3 Getting Started - What is Git?

* Gitillä voi tallentaa muutokset lokaalisti offline tilassa ja puskee myöhemmin main branchille kun oot netissä.
* Git tallentaa objektit tietokantaansa sisällön hashin perusteella, ei tiedostonimen perusteella.
  * NOTE! Jos sinulla on kaksi täsmälleen samansisältöistä tiedostoa eri nimillä ja eri kansioissa, Git tallentaa sisällön vain kerran. Molemmat "tiedostot" osoittavat samaan objektiin tietokannassa.

##    Gitin käyttö on lähinnä 'git add --all && git commit; git pull && git push'. Selitä tuon komennon jokainen osa. Käytä apuna itse valitsemiasi lähteitä ja viittaa niihin.

* && = " jos edellinen komento onnistui"
* ; = "joka tapauksessa suoritettava"
* git add = siirtää muutoksia työhakemistosta (working tree) staging-alueelle [(valmistelu seuraava comittia varten)](https://git-scm.com/docs/git-add)
  *  Lippu --all (myös -A) lisää indeksiin kaikki muutokset koko repositoriossa
*  commit ottaa "tiedostot sellaisina kuin ne ovat staging-alueella ja tallentaa snapshotin pysyvästi Git-hakemistoosi."
  * Commit on paikallinen
  * commit luo uuden objectin jolla on oma tiiviste
* git pull = hakee remote-repositoryn viimeisimmät commitit paikalllisesti ja yhdistää nuo haetut muutokset paikalliseen branchiin
* git push = on tapa synkronoida paikalliset commitit remote-repositoryyn
